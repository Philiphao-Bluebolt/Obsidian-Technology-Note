## C++

### 误操作导致 size() 方法返回的无符号整型数溢出

+ **空容器`size()`减一导致数值溢出**：由于容器的`size()`方法返回值默认为无符号整型，当容器大小为空时，减一变负值会导致无符号整型数溢出。正确写法是用`<=`、`>=`代替`<`、`>`

```cpp
if (i > arr.size() - 1)
// 改为
if (i >= arr.size())
```

### 误使用函数 to_string() 转换字符

+ **字符使用`to_string()`误转为ASCII码**：函数`to_string`会把单个字符转换为对应的ASCII码字符串而非直接转换为字符串；字符转字符串的正确方法是在空字符串后连接字符

```cpp
char c = '1';
str = to_string(c);
// 改为
str = std::string() + c;
// 或
std::string str(1, c);
```

### 误以为gcc编译器默认检查容器越界访问

+ **数组越界`gcc`不报错**：`gcc`编译器默认不检查容器访问越界等未定义行为；若要加上这个功能，需要在编译时调用模块`-fsanitize=address,undefined`，否则只能插桩调试

```bash
g++ -std=c++17 -g -fsanitize=address,undefined -O1 main.cpp -o main
```

## Python


### 创建二维数组时列表误复制引用


+ **创建二维数组时列表误复制引用**：在Python中用乘号`*`扩展列表时默认为浅拷贝，即对列表中的**可变对象**（如列表、字典、集合、自定义对象）仅复制其引用而非创建一个相同的全新对象。因此用列表创建二维数组时必须使用`for...in`语法。

```python
m, n = 3, 2

# 错误写法，第二维度的几个元素是指向同一列表的引用
my_matrix = [[0] * 2] * 3 

# 正确写法
my_matrix = [[0] * 2 for _ in range(3)]
```

### 误将除法运算符用作整除

+ **误使用除法运算符`/`进行整型除法**：Python与C++的自动类型转换规则不同，使用除号`/`进行整型数除法时，若除不尽，Python会自动将输出转换为浮点数，而并非像C++自动向下取整。Python中的整型除法运算符应为`//`

```python
a = 5
b = 2
print(a / b) # 2.5
print(a // b) # 2
```

### 误以为链式赋值是从右到左解析

+ **误使用链式赋值延长链表**：与C语言不一样，Python的链式赋值解析顺序本质上并非从右到左，因此不能用来连续赋值

```python
# 错误写法
n = n.next = ListNode()
# 等价于
temp = ListNode()
n = temp
n.next = temp
# 正确写法
n.next = ListNode()
n = n.next
```

### 尝试读取内操作方法的返回值

+ **尝试获取`list.reverse()`的返回值**：Python列表的`reverese()`方法对列表进行原位翻转，并无返回值。实际上，大部分列表操作方法均无返回值，`pop()`除外

```python
# 错误写法
return arr.reverse()

# 正确写法
arr.reverse()
return arr
```
