+ 🗂️ **数据结构选用**
	
	+ **哈希表 (Hash Table)**：适用于需要在常数级时间复杂度之内执行快速查询的任务，缺点是无法记录元素之间隐含的顺序关系。
	
	+ **链表 (Linked List)**：适用于需要频繁在序列中间删插元素的任务，缺点是访问任意节点的时间复杂度为线性级；若需要频繁获取任一节点前后的信息，则应使用双向链表。
	
	+ **队列 (Queue)**：适用于需要记录元素先后顺序，且没有中间删插或读取需求的任务。若需要在队首和队尾同时进行删插操作，则应使用双端队列。

## ⛓️‍💥 链表 Linked List


+ 📜 **常见题型**
	
	+ **链表重排 (List Reorder)**：按照某种规则重新排列链表的节点，如奇偶节点分离、数值小于n的节点前移；一般要求作答算法的空间复杂度为$O(1)$，时间复杂度为$O(n)$。

+ ✨️ **实用技巧**
	
	+ **假节点 (Dummy Node)**：用一个临时节点作为待处理链表的头节点`head`，可以避免一些边缘案例（如节点为`None`造成越界访问），若处理双端链表则可能头尾都要加假节点。
	
	+ **龟兔算法 (Hare and Tortoise Algorithm)**：即[Floyd算法](弗洛伊德算法%20Floyd's%20Algorithm.md)，快慢指针同时出发，快指针`fast`一次跳两步，慢指针`slow`一次跳一步，可用于检测链表环路或链表中间节点。	
	
	+ **双向链表 (Doubly Linked List)**：每个节点多存一个指向上一个节点的指针，适用于经常需要从链表中间节点回溯的问题，如节点移动或重排。
	
	+ **链表倒序 (Reverse of Linked List)**：对于操作单个节点的题目，如果操作条件与节点后面的信息有关，可能需要考虑将单向链表部分或全部倒序。
	
	+ **带节点哈希表的链表 (Linked List with Node Hashing)**：将链表与哈希表结合，所有节点存入哈希表，从而实现$O(1)$复杂度的任意节点访问。
