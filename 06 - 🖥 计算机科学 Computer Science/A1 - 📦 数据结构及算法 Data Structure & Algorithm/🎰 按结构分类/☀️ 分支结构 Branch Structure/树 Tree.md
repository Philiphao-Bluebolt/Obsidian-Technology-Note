## 基本概念 Basic Concept

树是由节点和边组成的数据结构，每一棵含$n$个节点的非空树都有$n-1$条边和唯一的根节点。从图的角度看，树是一种带层次结构的无向无环连通图。

```
        1(1) 
       /    \
    2(9)     3(5)
   /    \        \
4(6)     5(5)     6(4)
             \
              7(1)
```

### 节点的类型与关系 Vertex Types and Relationship

节点是组成树结构的元素，也称为顶点、结点或Node，每一个节点之下可以连接多个子节点或子树，除根节点外，所有节点之上有且只有唯一一个亲代节点。

> **根节点 Root** - 任何非空树有且只有一个根节点，位于顶层。根节点没有亲节点。

> **叶节点 Leaf** - 没有子节点的末端节点

> **亲代节点 Parent** - 亦称双亲节点（此名称的“双”字有误导性），一个节点之上直接连接的节点

> **孩子节点 Child** - 一个节点之下直接连接的子节点

> **兄弟节点 Sibling** - 亲代节点相同的多个节点之间互为兄弟节点。

> **祖先节点 Ancestor** - 一个节点的祖先节点是从根节点开始到该节点的路径上的所有节点。根节点是所有其他节点的祖先节点。

> **后代节点 Descendant** - 亦称子孙节点，一个节点的后代节点是该节点之下连接的所有子树之中的节点。所有其他节点都是根节点的后代节点。

> **子树 Subtree** - 一个节点的子树即以它为根的树，一棵树的子树包括其所有节点的子树

> **层 Level** - 节点的层数等于从根节点到该节点的边数（根节点0层）或边数+1（根节点1层）。树的**深度**即为最深层数。

> **度 Degree** - 节点的度是其后代节点的个数或子树的个数，树的度是所有节点之中的最大度。


### 树的类型 Tree Types


+ **空树 Empty Tree** - 节点数和边数为零的树。节点数不为零的树为非空树。

+ **有序树 Sequential Tree** - 兄弟节点之间存在先后顺序的节点。兄弟节点之间不存在顺序的节点称为无序树。

+ **二叉树 Binary Tree** - 最常用的树结构之一，每个节点至多有两个孩子节点的树。一棵$n$层的二叉树最多有$2^n-1$ 个节点。二叉树区分左右孩子节点，可能出现左孩子节点为空，右孩子节点不为空
	+ **满二叉树 Full Binary Tree** - 所有非叶子节点都有两个孩子节点的二叉树
	+ **完全二叉树 Complete Binary Tree** - 除最后一层以外所有层都填满，最后一层节点左边连续填满，右边连续缺失或填满的二叉树。一颗$n$层的完全二叉树节点数介于与$2^{n-1}$与$2^n-1$ 之间。
	+ **完美二叉树 Perfect Binary Tree** - 拥有$2^n-1$个节点的二叉树，所有叶子节点均在同一层且没有孩子节点，所有非叶子节点都有两个孩子节点


+ **搜索树 Search Tree** - 搜索树是一类用于快速搜索的树，如二叉搜索树、B树，其特点是每个节点与其后代节点的值必须满足一定的顺序关系

### 树的遍历 Traverse a Tree

树的遍历是按一定顺序将原本非线性的节点结构排列成线性结构的过程，遍历算法同时也可用于搜索。

+ **深度优先搜索 DFS** - 优先往深处遍历，先抵达叶节点，再访问其兄弟节点，最后返回上层访问其他兄弟分支，直到所有子树都被访问完。（王位继承顺序）

+ **广度优先搜索 BFS** - 优先访问同一深度的所有兄弟节点或堂兄弟节点，然后再按兄弟节点顺序访问其他子树。（文件搜索顺序）

+ **前序搜索 Preorder** - 又称为**先根搜索**，先访问根节点，再访问子树

> **中序搜索 Inorder** - 只适用于二叉树，先访问左节点，再访问根节点，最后访问右节点。

> **后序搜索 Postorder** - 又称为**后根搜索**，先访问子树，再访问根节点

### 链表储存 List Storage

以结构体或类的形式定义节点，将节点值与指向亲代、孩子或兄弟节点的**指针**封装在一起，类似多指向的链表。树的链表储存方法根据指针指向有不同的变体。

+ **孩子标识法** - 每个节点记录其孩子节点的指针，普通树直接用向量存放指针，二叉树用两个指针即可区分左右孩子。缺点是寻找指定节点的亲代需要遍历整棵树。

```cpp
struct TreeNode{
	int val;
	vector<TreeNode*> children;
};
```

+ **孩子亲代标识法** - 每个节点同时记录其孩子节点和亲代节点，类似双向线性链表。优点是容易找到任一节点的亲代和孩子，缺点是占用空间大。

```cpp
struct TreeNode{
	int val;
	TreeNode *parent;
	vector<TreeNode*> children;
};
```

+ **长子幼弟标识法** - 亦称孩子兄弟表示法，所有节点只记录指向自己的长子（最左边第一个孩子节点）和幼弟（右边的下一个兄弟）的指针

```Cpp
struct TreeNode{
	int val;
	TreeNode *first_child;
	TreeNode *next_sibling;
};
```

### 文本储存 Text Storage

先按深度优先从左到右的顺序给所有节点编号，第一行记录节点数，第二行记录节点的值，后面各行则用不同方法记录节点的连接结构

+ **记边法** - 后面$n-1$行用亲代-孩子的格式记录边，先出现的兄弟节点靠左，适用于不区分左右孩子的普通二叉树。表示二叉树时，遇到只存在右孩子节点的情况需要特殊标记（如用0标识）

```
7

1 2 
1 3

2 5
0 3 6
0 5 7
```

+ **记子法** - 后面$n$行分别表示每个节点的孩子，可用于记录二叉树

```
7
1 9 5 6 5 4 1
2 3 
4 5 
0 6
0 0
0 7
0 0
0 0
```

+ **中序法** - 第一行记录节点数$n$，第二行记录节点的值，后面按中序遍历的思想记录

```
7
1 9 5 6 5 4 1
0 5 7
4 2 5
2 1 3
0 3 6
```

### 数组储存 Array Storage

二叉树特有的储存方法，按照**完全二叉树**的节点编号将目标树节点的值依次填入数组对中，会产生较多空位，因此空间利用率低。适合存放完全二叉树。

```
1 9 5 6 5 4 - - - 1
```

+ **空位优化** -  出现连续空位时用数字表示空位个数，减少空位数量。

```
1 9 5 6 5 4 - 3 1
```
