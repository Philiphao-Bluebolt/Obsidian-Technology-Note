## 系统功能


### 蓝牙无法连接Airpod

> **系统** - Ubuntu 22.04 双系统
> **具体问题** - 蓝牙可以搜索到Airpod，但无法正常配置与连接，设备一栏显示“Not set”（未设置），点击连接无反应

> **解决方案** - [参考](https://gist.github.com/aidos-dev/b49078c1d8c6bb1621e4ac199d18213b)

打开蓝牙配置文件`sudo vim /etc/bluetooth/main.conf`，修改以下部分

```
#ControllerMode = dual
```

改为

```
#ControllerMode = dual
#ControllerMode = bredr
```

保存配置文件，重启蓝牙模块

```
sudo /etc/init.d/bluetooth restart
```

### 磁盘变为只读

> **系统** - Ubuntu 22.04 双系统
> **具体问题** - 磁盘变为只读挂载，对磁盘文件的修改无法正常保存，即使重启系统或者手动重新挂载也无用

原因是Windows系统没有正常关闭，硬盘文件的格式异常

解决方案就是重新启动Windows系统，等待所有系统更新完成，正常关机，再启动Ubuntu系统

## 软件安装


### Apt无法连接ROS源

> **具体问题** - `apt`无法连接ROS官网的插件仓库`package.ros.org`

```bash
Cannot initiate the connection to packages.ros.org

W: An error occurred during the signature verification. The repository is not updated and the previous index files will be used. GPG error: http://packages.osrfoundation.org/gazebo/ubuntu-stable focal InRelease: The following signatures couldn't be verified because the public key is not available: NO_PUBKEY 67170598AF249743

```

**解决方案**

+ 更新`apt-key`的ROS源密钥
+ 开启VPN
+ 换源（据说换源也是导致密钥出问题的原因）

## 虚拟机


### 虚拟机无法连接网络

> **具体问题** - 右上角网络标志显示为问号

**解决方案：**

+ 重启宿主机和虚拟机（简单粗暴）
+ 关闭虚拟机VPN网络代理
+ 重启网络服务

```
nmcli n off
nmcli n on
```

+ 关闭隐私的连接性检查


### 虚拟机复制粘贴不起作用

+ 虚拟机 --> 主机不起作用，试试连续按两次`Ctrl` + `C` 
+ 主机 --> 虚拟机不起作用，记住不能直接复制到桌面，一定要复制到打开的文件夹窗口里
